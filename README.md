# Viettel Medical IE Pipeline

Hybrid rule, terminology, NER, and LLM pipeline for Round 1 of the Viettel AI Race Medical Challenge.

## Setup

```bash
pip install -r requirements.txt
```

## Data

Place input `.txt` files in:

```text
data/input/
```

Optional terminology:

```text
data/terminology/rxnorm_2026/
data/terminology/icd10_vi/
data/terminology/snomed/
data/terminology/custom_aliases/
```

The repository includes small local seed files for conservative candidate
linking:

```text
data/terminology/custom_aliases/rxnorm_seed.tsv
data/terminology/icd10_vi/icd10_vi_seed.tsv
```

The RxNorm seed was generated from the official NLM RxNav API current version
`06-Jul-2026`. Full RxNorm RRF files are not bundled; if available, place
`RXNCONSO.RRF` in `data/terminology/rxnorm_2026/`.

## Run Inference

The default configuration expects an OpenAI-compatible server serving
`Qwen/Qwen3-8B` at `http://127.0.0.1:8000/v1`. The request disables thinking
mode and asks for JSON output. Change `models.llm.base_url` when the server is
remote, and set `LLM_API_KEY` when authentication is required.

Active model budget:

```text
Qwen3-8B:              8.200B
XLM-R VietMed-NER:     0.278B
Total:                 8.478B / 9.000B
```

PhoBERT VietMed-NER is disabled while this hybrid is active. The LLM extracts
entity text, type, and assertion only. Offsets are validated against the input,
and ICD-10/RxNorm candidates are generated by the local terminology linker.
If the LLM server is unavailable, it is disabled after the first failed request
and the local rule/NER pipeline continues.

```bash
bash run_inference.sh data/input output.zip
```

If your files are in the extracted challenge folder, use:

```bash
bash run_inference.sh input output.zip
```

## Output

Creates:

```text
output/
output.zip
```

Each record JSON is a list of extracted spans:

```json
[
  {
    "text": "metoprolol 25mg po bid",
    "position": [76, 98],
    "type": "THUỐC",
    "assertions": ["isHistorical"],
    "candidates": ["Rx..."]
  }
]
```

`position` uses Python-style character spans: `[start, end)`, where `end`
is exclusive.

## Hybrid Design

```text
Qwen3 entity/type/assertion ----+
VietMed-NER boundary signals ---+--> span merge --> assertion scope repair
Rules and terminology ----------+                         |
                                                          v
                                               ICD-10/RxNorm linker
```

LLM output never supplies ICD-10 or RxNorm identifiers. This avoids generated
codes and keeps candidate mapping reproducible from local terminology data.

## Reproducibility

All terminology is loaded locally when present. LLM inference uses the server
configured in `configs/default.yaml`; the model weights are not stored in this repository.
RxNorm should use a 2026 release snapshot under `data/terminology/rxnorm_2026/`.
Diagnosis/problem linking should use Vietnamese ICD-10 terminology under `data/terminology/icd10_vi/`.
